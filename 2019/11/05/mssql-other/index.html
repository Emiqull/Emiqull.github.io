表概念

  • 属性 = 字段 = 域 =列
  • 元组 = 记录 = 行
  • 二维表 = 关系
  • 关键字 = 主键

范式

  • 第一范式：元组中的每个属性都是原子数据项，不可再分割
  • 第二范式：非关键字属性完全依赖关键字属性
  • 第三范式：非关键字属性间没有传递关系
  • 第四范式：属性的若干个值由另一个属性的若干个值决定，一个关系中最有多有一个存在
  • 第五范式：一个表不能分割成多个表而不丢失信息

COUNT(*)查询优化

若必须对某张表多次使用count(*)操作时，在该表中的一个数据类型所占字节最少的列上建立索引，可以提升count(*)的查询性能

多表连接优化

两个或两个以上的表进行join连接查询时：

  • 在连接后的ON限制字段上建立索引
  • 连接后的ON限制字段是已存在非聚集索引的第一个字段

通配符

  • %：匹配任意个数的字符
  • _：匹配一个字符
  • [charlist]：匹配字符列表中的任意一个字符
  • [^charlist]或[!charlist]：不匹配字符列表中的任意一个字符

BETWEEN…AND

  • 该子句操作的数据包括：数值、日期、文本(默认比较文本的第一个字母的ASCII码值，若相同则比较下一个字母 … )
  • 如果需要操作该子句范围之外的数据，在BETWEEN…AND前面加NOT 即可

编写T-SQL的小技巧

  • 查询语句中避免使用SELECT * 查找不需要的列会产生不必要的IO
  • 声明变量时指定长度 不指定变量的长度，可能会因为默认的长度只有1，导致数据内容的丢失
  • 使用合适的变量类型 比如不使用字符串类型存储日期数据，首先是性能不好，其次是内置的时间函数无法使用
  • 使用SCHEMA前缀来选择表 指定Schema.Table的方式避免了无谓的解析，提高执行T-SQL的效率
  • 使用合理的命名规范命名 建议使用实体对象+操作类型，比如Product_Update
  • 插入大量数据
    • 方式一：如果要使用循环，要放到一个事务中
    • 方式二：使用批量插入，即INSERT INTO … SELECT …
    • 方式三：使用CTE方式批量插入，即WITH cte AS(SELECT …) ; INSERT INTO … SELECT … FROM cte
    • 注：插入时间效率依次增高
  • where条件子句使用建议
    • 减少使用数据类型转换
    • 避免对字段进行NULL值的判断
    • 避免使用OR连接条件，建议改为UNION ALL合并
    • 避免模糊查询使用前置%
    • 避免使用参数
    • 避免对字段进行表达式操作，如(WHERE num/2=100改为WHERE num=100*2)
    • 避免对字段进行函数操作，如 改为
    • 避免使用!=和<>操作符，数据库会对< 、 <= 、 = 、 > 、 >= 、 BETWEEN AND使用索引，如 改为

    • SET NOCOUNT 在所有的存储过程和触发器的开始处设置SET NOCOUNT ON、在结束时设置SET NOCOUNT OFF； 系统默认SET NOCOUNT OFF

链接服务器

在两个不同的SQL SERVER 的服务器之间进行数据访问和更新，建立步骤如下：

  • 【服务器对象】-【链接服务器】-右键【新建链接服务器】
  • 【常规】选项卡 >> 输入【链接服务器地址】、选择SQL Server【服务器类型】(根据实际数据库类型选择服务器类型)
  • 【安全性】选项卡 >> 选择【使用此安全上下文建立连接】，设置登录用户和密码(为了添加一个本地服务器链接到远程服务器的登录映射)
  • [【服务器选项】选项卡>>RPC和RPC Out选择True；]
  • 退回到【链接服务器】，选择刚刚建立的链接服务器，右键【测试连接】
  • 测试连接成功，即完成链接服务器建立

注：通过链接服务器不能删除(including truncate)链接源服务器的任何对像

  • 通过链接服务器可以对链接源服务器的表进行insert,updae,delete操作.http://www.cnblogs.com/xcloudbiz/articles/4923018.html

聚集索引和非聚集索引

  • 聚集索引一个表只能有一个，而非聚集索引一个表可以有多个
  • 聚集索引存储记录是物理上连续存在的，而非聚集索引是逻辑上连续的，物理存储并不连续
  • 聚集索引可以创建在任何一列的字段上，但非常不建议这样做
  • 如果未使用UNIQUE属性创建唯一聚集索引，数据库引擎将向表自动添加一个四字节uniqueifer列，必要的时候，数据库引擎将向行自动添加一个uniqueifier值，使得每个键唯一。该列和；列值供内部使用，用户不可查看或访问
  • 主键是在查询的时候定义到表记录的一行，而非聚集索引在查询范围的时候性能更好
  • 索引是通过二叉树的形式进行描述的，聚集索引的叶子节点就是数据节点，而非聚集索引的叶子节点仍是索引节点，但该索引节点有一个执行最终数据的指针
