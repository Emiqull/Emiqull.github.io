概述

  • 触发器是在对表进行插入、修改和删除的操作时自动执行的存储过程，触发器是一种特殊类型的存储过程
  • 触发器是一种高级约束，可定义比check约束更为复杂的约束(执行复杂的SQL语句:if,while,case；可引用其他表中的列)
  • 本身就是一个事务，若执行出现错误，自动回滚
  • 触发器和普通的存储过程的区别是：触发器是当对某一个表进行操作，如：update、insert、delete这些操作的时候，系统会自动调用执行该表上对应的触发器

分类

  • INSERT触发器、UPDATE触发器、DELETE触发器，都属于after触发器（之后触发） ，且只能定义在表上
  • INSTEAD OF 触发器 （之前触发）表示仅执行触发器本身，可以定义在表和视图上

INSERTED表和DELETED表

摘要：

  • 触发器在触发时有两个特殊的表：插入表（instered表）和删除表（deleted表），系统在内存中自动创建INSERTED表和DELETED表
  • 两张表都是只读属性，触发器执行完成后自动删除。这两张表的结果总是与被触发器应用的表结构相同

INSERTED表：临时保存执行插入或更新后的表记录

  • 可以从INSERTED表中检查插入或更新后的数据是否满足需求，若不满足，则报告错误信息并自动回滚插入或更新操作

DELETED表：临时保存执行删除或更新前的表记录

  • 可以从DELETED表中检查被删除的数据是否满足需求，若不满足，则报告错误信息并自动回滚删除操作

注解：定义触发器的表执行记录操作，与插入insert表和删除delete表的关系及其存储内容,比较列表如下：

| 操作 | Inserted逻辑表 | Deleted逻辑表 |
| :————: |:—————:| :—–:|
| 增加记录(insert) | 存放增加的记录 | 无 |
| 删除记录(delete) | 无 | 存放被删除的记录 |
| 修改记录(update) | 存放更新后的记录 | 存放更新前的记录 |

注意：UPDATE数据时，就是先删除表记录，再插入一条记录
