Q&A / DELETE 和 TRUNCATE

文章目录

语法格式
语法区别
语法应用

语法格式

  • DELETE格式：DELETE FROM 表名 [WHERE … ];
  • TRUNCATE格式：TRUNCATE TABLE 表名;

语法区别

  • TRUNCATE比DELETE执行删除操作，使用的事务日志空间更少

    • DELETE是一行一行的删除表记录，并把每一次的删除操作都记录到日志文件
    • TRUNCATE删除，是以页为单位进行操作，在日志文件的记录中，仅仅记录释放每一页的动作，不会记录每一行的操作

  • TRUNCATE比DELETE执行删除时，使用的锁要少

    • DELETE由于是一行一行的删除，所以需要对处理的行进行加锁，而且是行锁；TRUNCATE操作由于是对页操作，所以只需要页锁或表锁

  • TRUNCATE删除表是清空表记录，DELETE删除表是删除记录中的内容

    • TRUNCATE删除表只可以删除整张表
    • DELETE删除表数据后，表中会存在占用空间的空白页(若表存在聚集索引，删除后只剩下一个页和一个区)，空白页会在新数据的插入后重用
    • TRUNCATE删除表中所有的记录后，会同时清空所有的空白页，但是会保留表的结构、索引、约束等

语法应用

  • 为了更好的删除表后的占用空间，建议在表中创建聚集索引
  • 若确定所有的表数据都不需要，建议先使用TRUNCATE删除表记录后，再使用DROP TABLE删除表
