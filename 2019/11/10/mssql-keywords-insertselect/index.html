INSERT INTO 和 SELECT INTO

语法格式

  • INSERT INTO … SELECT … ：INSERT INTO TargetTable[(field … )] SELECT … ;
  • SELECT …INTO ：SELECT field … INTO TargetTable FROM SourceTable;

语法区别

  • INSERT INTO … SELECT … 语句要求目标表必须已经存在，SELECT …INTO 语句要求目标表必须不存在

    • INSERT INTO … SELECT … 语句中，因为目标表已经存在，所以执行时还可以向表中插入常量
    • INSERT INTO … SELECT … 语句中，若目标表中存在主键，则字段中一定要存在主键字段
    • INSERT INTO … SELECT … 语句中，目标表与源表的表结构要相同(数值的varcahr类型转换成int类型可以成功解析)

  • INSERT INTO … SELECT … 语句是循环查询结果集的每行，插入到已存在的表中；SELECT …INTO 语句是把查询结果集直接保存成表

    • SELECT …INTO 语句在执行时，会自动创建目标表并插入数据

语法应用

  • 若需要对表的数据进行备份，且追求效率，建议使用SELECT …INTO 语句
