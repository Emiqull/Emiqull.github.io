MERGE简述

  • MERGE一个DML关键字,它在SQL Server 2008被引入，它能将Insert,Update,Delete的功能合并为一句
  • MSDN对于Merge的解释:”根据与源表联接的结果，对目标表执行插入、更新或删除操作。”

例如

  • 根据源表在目标表中找到存在差异的记录，在目标表中插入、更新或删除行，并可选择的对源表操作(即可以对两个表进行同步)
  • 需要使用Merge的场景比如:数据同步、数据转换和基于源表对目标表做Insert,Update,Delete操作

示例解析

目的:创建一个源表和目标表，根据源表的记录对目标表的记录进行操作

  • 若源表与目标表的id相同，则更新目标表的记录为源表的记录
  • 若目标表不存在源表的记录，则向目标表中插入源表的记录
  • 若目标表中存在源表中没有的记录，则删除该仅在目标表中存在的记录

MERGE语句Tips

  • 执行MERGE语句一次只能操作一个目标表
  • 执行MERGE语句的源表中若有重复的记录，则只允许执行INSERT语句(即第四点中满足NOT MATCHED关键字，也就是不满足匹配条件)插入所有重复记录
  • 在MERGE语句的MATCHED关键字之后，只允许执行一次UPDATE语句或一次DELETE语句
  • 在MERGE语句的NOT MATCHED关键字之后，只允许执行INSERT语句
  • MERGE语句的结束必须以英文分号“;”结尾
  • 在MERGE语句中默认关键字MATCHED关键字说明的对象是目标表，但可以在MATCHED关键字之后加上BY Target或BY Source来显式说明MATCHED的对象
